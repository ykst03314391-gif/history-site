# 戦国時代 情報サイト（仮）

日本の戦国時代（おおよそ 15 世紀末〜16 世紀末）を、**管理人が自分の視点で捉え直す個人メディア**。
考察（記事）を主役に据え、人物・合戦・城・史跡のデータベースはそれを支える「脚注」として構造化する。事実は出典に基づき、そのうえで「自分はこう考える」という態度を前面に出す。

---

## コンセプト（背骨）

**戦国時代を、自分の視点で捉え直す個人メディア。**

このサイトを貫く「串」は、テーマでもジャンルでもなく、**管理人という一人の視点**である。
歴史上の事実は出典に基づいて正確に扱う。そのうえで、「自分はこう捉える」という**態度**を前面に出すことが、このサイトの存在理由であり、最大の差別化。

事実だけなら Wikipedia がある。ここにしかないのは、**一人の人間が戦国時代をどう見ているか**という視点そのもの。

### 一貫する態度：「別の角度から捉え直す」

通説をなぞらない。教科書の順番で並べない。
**「本当にそうか？」「別の見方はないか？」** と問い直し、自分なりの答えを出す——この態度が全コンテンツに共通する背骨。

特集はこの態度の「現れ」であって、それぞれ別テーマではない。同じ一人の視点を、違う角度から当てているだけ。

| 特集 | 当てている角度 |
|---|---|
| **経済で見る戦国時代**（economics） | 大名を「経営者」に見立て、知行・恩賞・収入という**カネの角度**から捉え直す |
| **社会の通説っておかしくね？**（myth-busting） | 広まった通説を、資料に基づいて**疑いの角度**から捉え直す |
| **戦国ゆかりの地めぐり**（旅・travel） | 机上の知識を、実際に立つ**現地の角度**から捉え直す |

> どの特集も、やっていることは同じ：**「別の角度から、自分の目で捉え直す」**。
> 新しい特集を足すときも、この態度に接続できるか（＝一人の視点の現れになっているか）を基準にする。

### 考察が主役、データベースは脚注

このサイトの主役は**考察（記事）**。人物・合戦・大名家のデータベースは、主役をやめて**考察を支える「脚注」** の役割に置く。

- 記事を読んでいて「この人物、どんな人だっけ？」となったとき、すぐ引ける**足場**としてDBがある。
- DBは「網羅」を目指さない。**考察が必要とする範囲**を、必要な深さで整える。
- 入口も「調べる」より「読む・捉え直す」を先に置く。

> 従来は「DB（主）＋記事（従）」で考えていたが、これを反転させる。
> **視点のない事実の集積は Wikipedia の劣化版**にしかならない。主役は常に「管理人の視点」。

### このサイトのスタンス（明示することが差別化）

差別化の核は、機能でも網羅性でもなく、**スタンスをはっきり掲げること**。

- 事実は出典に基づく。諸説あるものは諸説あると書く。
- そのうえで、**「自分はこう考える」を隠さない**。中立を装わない。
- 通説・定説・ゲームの評価であっても、資料に照らして**再評価する**（例：「信長の野望、実は間違ってる？」）。
- 読者を「調べて終わり」にせず、**読む本・行く現地**という次の行動へつなぐ。

> 常に自問する：これは「事実の再掲」に留まっていないか。**自分の視点は、どこに出ているか。**

## Wikipedia との違い（従来の整理）

> 上の「背骨」がこのサイトの立脚点。以下は、その背骨を「Wikipedia と何が違うのか」の観点で言い換えた補足。

- Wikipedia は「**引く**（調べて確認する）」ための、中立・網羅の事典。
- 本サイトは「**捉え直す**（一人の視点で読み解く）」ための、態度を持った個人メディア。
- 事実部分は重なる。差が出るのは **切り口・考察・スタンス**。網羅性では勝負しない。

## 主要コンテンツ・機能

### A. 人物データベース（「誰だっけ？ どんな人だっけ？」に答える）

**「誰だっけ？／とは？」（基本情報）**
- 名前、呼名（幼名・通称・法名など）、官位
- 生没年、生まれた土地
- 親・子・親戚（**家系図**）
- 家紋
- 人物画像
- 家臣団、仕えた人（主従関係）

**「どんな人だっけ？」（人物像）**
- 年表（個人の生涯）
- 領地変遷
- 逸話・エピソード
- 成し遂げたこと

**「もっと知りたい！」（深掘り・現地情報）**
- 参考書籍の紹介
- 関連する博物館・資料館
- ゆかりの名物
- 城跡・銅像・刀剣

### B. 地域から探す（「この土地には何がある？」）

ある県・市を訪れたときに、その土地の歴史がわかる。
- 都道府県別の年表・出来事
- その土地を治めた人物の変遷
- 古戦場
- 城

### C. テーマ別まとめ

- 大名家別の変遷
- グループまとめ（武田十六将、徳川四天王 など）
- 有名な合戦
- **逆引き検索**：調べたい人物 → 全国どこの史跡・博物館へ行けばよいか
  （例：豊臣秀長 → 奈良の史跡、竹田城、秀吉関連の展示がある博物館 など）
- 世代別比較（同時代・同世代に誰がいたか）
- 官位変遷

### D. 研究・新説の追跡（メディア面）

- 「あの通説、実は違った？」
- 「こんな新発見があった！」
- 上記に対する自分の意見・考察

### E. コラム / 記事（発信）

- トレンドに合わせた記事（大河・新作映画 絡み）
- 新説に対する自分の意見
- 旅・史跡めぐりの感想記事

### F. 考察・企画もの

- 「信長の野望、実は間違ってる？」
  ゲーム内の評価が低い／高い武将の、実像に基づく再評価まとめ

### G. クイズゲーム（エンタメ）

- 戦国時代に関するクイズを内包（検討中）

### H. オリジナルキャラクター

- 自作キャラ「ぱぱねこ」の派生キャラを制作
  - 特徴的な兜をかぶせるなど、武将モチーフ化
- サイトのマスコット・親しみやすさに活用

## 運用方針

- **出典重視**：掲載情報は可能な限り出典を明記する。諸説ある場合はその旨を記載
- **継続更新**：新説・新研究に応じて内容をブラッシュアップしていく

> ⚠️ **現状のDBデータは全て「仮置き」**：人物・合戦・大名家の各データは、いまのところ管理人（および生成）の一般知識で書いた暫定値で、**記載中の出典（信長公記など）は未検証のラベルにすぎない**。一次資料での裏取りは未実施。DB系ページ（`person`/`people`/`battle`/`battles`/`clan`/`clans`/`regions`）には「仮置きデータ」注記を表示している。順次、原典に当たって修正し、検証済みになった記述から `sourceId` を正しく紐付けていく。

### 編集ルール：大名家の「一門」の範囲

大名家ページで「一門」と「ゆかりの人物（家臣／その他）」を分類する際の基準。
判断軸は「**その家に入るか／家から出るか**」。

**一門に含める**
- 当主の血族（父母・兄弟姉妹・子・孫・甥姪 など）
- 家に入った近親：正室、迎えた養子、婿養子（娘婿）
- 他家へ養子に出た実子（血がつながるため。「◯◯家を継ぐ」と注記する）

**一門に含めない**
- 他家へ嫁いだ実娘（婚姻で嫁ぎ先の家人となる＝姻戚。その人物ページの家族欄には残すが、実家の大名家ページの一門には載せない）
- 血縁も姻戚もない家臣（→「家臣」に分類）

> ポイント：娘婿・養子は「家に入る」ため一門。実娘を嫁がせるのは「家から出る」ため一門としない。
> 詳細は [`docs/schema-clan.md`](docs/schema-clan.md) を参照。

## ローカルでの起動方法

データを `fetch` で読み込むため、**HTML ファイルを直接ダブルクリック（`file://`）で開くと動作しません**。
簡易 HTTP サーバー経由で表示してください。

プロジェクトのルート（`history-site/`）で、以下のいずれかを実行します。

```bash
# Python がある場合
python -m http.server 8765

# Node.js がある場合
npx serve -l 8765
# もしくは
npx http-server -p 8765
```

その後、ブラウザで以下を開きます。

- 一覧ページ： http://localhost:8765/index.html
- 人物詳細（例）： http://localhost:8765/person.html?id=oda-nobunaga

停止する場合はサーバーを起動したターミナルで `Ctrl + C`。
