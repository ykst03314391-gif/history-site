# 戦国時代 情報サイト（仮）

日本の戦国時代（おおよそ 15 世紀末〜16 世紀末）を対象とした、**データベース型の歴史情報サイト**。
人物・合戦・城・史跡などを構造化して相互参照でき、最新の研究・新説を反映しながらブラッシュアップしていくことを目指す。

---

## コンセプト

大河ドラマ・映画・ゲームなどである人物が出てきたとき、
「**これ、誰だっけ？ どんな人だっけ？ もっと知りたい！**」
という疑問に、**ちゃんとした出典**に基づいて答えるサイト。

「なんとなく知ってる」レベルの知識を、体系的に再学習・深掘りできる場にする。

### 3つの性格

1. **データベース** — 人物・合戦・城・史跡を構造化し、相互に関連付けて参照
2. **メディア / ブログ** — 最新研究・新説の追跡、自分の意見・考察、旅・史跡めぐりの記事
3. **エンタメ** — クイズゲーム、オリジナルキャラクターによる親しみやすさ

## Wikipedia との差別化方針

「これ、Wikipedia でよくない？」とならないために、本サイトは**網羅性で勝負しない**。
Wikipedia が構造上できない・やらないことに価値を置く。

### Wikipedia が「やらない／できない」こと ＝ 本サイトの強み

| 観点 | Wikipedia | 本サイト |
|---|---|---|
| **主観・考察** | 中立的観点・独自研究禁止のため、書き手の意見や新説への評価は載せられない | 出典を示したうえで、**管理人の考察・意見・新説への評価**を明確に打ち出す（例：「信長の野望、実は間違ってる？」） |
| **学習動線** | 網羅的な事典。目的別の導線はない | 「**誰だっけ？→どんな人？→もっと知りたい！**」という再学習の動線に特化した設計 |
| **場所からの逆引き** | 人物・事項ごとの記事が中心。「この土地には何がある」「この人物を調べるにはどこへ行けば」の横断は弱い | **地域から探す／逆引き検索**（都道府県別・史跡・博物館への送客）を一次機能にする |
| **横断・比較** | 記事間リンクはあるが、横断的な比較ビューはない | 構造化データで**世代別比較・官位比較・家臣団や大名家の関係**を横断表示 |
| **体験・エンタメ** | 事典であり、遊びの要素はない | **クイズ**やオリジナルキャラ「ぱぱねこ」で、親しみやすく・繰り返し訪れたくなる体験に |
| **次の行動へつなぐ** | 記事を読んで終わり | 参考書籍・史跡・展示へ誘導し、**「実際に読む・現地へ行く」次の行動**に接続 |

### 一言でいうと

- Wikipedia は「**引く**（調べて確認する）」ための事典。
- 本サイトは「**巡る・深める・遊ぶ**」ための、視点と体験を持ったメディア。
- 事実は出典に基づいて正確に。そのうえで**編集方針・切り口・意見**という「人格」を持たせることが最大の差別化。

> 事実部分だけを見れば Wikipedia と重なる。差が出るのは「切り口・導線・考察・体験」。
> 単なる事実の再掲に留まっていないか、を常に自問する。

## 主要コンテンツ・機能

### A. 人物データベース（「誰だっけ？ どんな人だっけ？」に答える）

**「誰だっけ？／とは？」（基本情報）**
- 名前、呼名（幼名・通称・法名など）、官位
- 生没年、生まれた土地
- 親・子・親戚（**家系図**）
- 家紋
- 人物画像
- 家臣団、仕えた人（主従関係）

**「どんな人だっけ？」（人物像）**
- 年表（個人の生涯）
- 領地変遷
- 逸話・エピソード
- 成し遂げたこと

**「もっと知りたい！」（深掘り・現地情報）**
- 参考書籍の紹介
- 関連する博物館・資料館
- ゆかりの名物
- 城跡・銅像・刀剣

### B. 地域から探す（「この土地には何がある？」）

ある県・市を訪れたときに、その土地の歴史がわかる。
- 都道府県別の年表・出来事
- その土地を治めた人物の変遷
- 古戦場
- 城

### C. テーマ別まとめ

- 大名家別の変遷
- グループまとめ（武田十六将、徳川四天王 など）
- 有名な合戦
- **逆引き検索**：調べたい人物 → 全国どこの史跡・博物館へ行けばよいか
  （例：豊臣秀長 → 奈良の史跡、竹田城、秀吉関連の展示がある博物館 など）
- 世代別比較（同時代・同世代に誰がいたか）
- 官位変遷

### D. 研究・新説の追跡（メディア面）

- 「あの通説、実は違った？」
- 「こんな新発見があった！」
- 上記に対する自分の意見・考察

### E. コラム / 記事（発信）

- トレンドに合わせた記事（大河・新作映画 絡み）
- 新説に対する自分の意見
- 旅・史跡めぐりの感想記事

### F. 考察・企画もの

- 「信長の野望、実は間違ってる？」
  ゲーム内の評価が低い／高い武将の、実像に基づく再評価まとめ

### G. クイズゲーム（エンタメ）

- 戦国時代に関するクイズを内包（検討中）

### H. オリジナルキャラクター

- 自作キャラ「ぱぱねこ」の派生キャラを制作
  - 特徴的な兜をかぶせるなど、武将モチーフ化
- サイトのマスコット・親しみやすさに活用

## 運用方針

- **出典重視**：掲載情報は可能な限り出典を明記する。諸説ある場合はその旨を記載
- **継続更新**：新説・新研究に応じて内容をブラッシュアップしていく

### 編集ルール：大名家の「一門」の範囲

大名家ページで「一門」と「ゆかりの人物（家臣／その他）」を分類する際の基準。
判断軸は「**その家に入るか／家から出るか**」。

**一門に含める**
- 当主の血族（父母・兄弟姉妹・子・孫・甥姪 など）
- 家に入った近親：正室、迎えた養子、婿養子（娘婿）
- 他家へ養子に出た実子（血がつながるため。「◯◯家を継ぐ」と注記する）

**一門に含めない**
- 他家へ嫁いだ実娘（婚姻で嫁ぎ先の家人となる＝姻戚。その人物ページの家族欄には残すが、実家の大名家ページの一門には載せない）
- 血縁も姻戚もない家臣（→「家臣」に分類）

> ポイント：娘婿・養子は「家に入る」ため一門。実娘を嫁がせるのは「家から出る」ため一門としない。
> 詳細は [`docs/schema-clan.md`](docs/schema-clan.md) を参照。

## ローカルでの起動方法

データを `fetch` で読み込むため、**HTML ファイルを直接ダブルクリック（`file://`）で開くと動作しません**。
簡易 HTTP サーバー経由で表示してください。

プロジェクトのルート（`history-site/`）で、以下のいずれかを実行します。

```bash
# Python がある場合
python -m http.server 8765

# Node.js がある場合
npx serve -l 8765
# もしくは
npx http-server -p 8765
```

その後、ブラウザで以下を開きます。

- 一覧ページ： http://localhost:8765/index.html
- 人物詳細（例）： http://localhost:8765/person.html?id=oda-nobunaga

停止する場合はサーバーを起動したターミナルで `Ctrl + C`。
