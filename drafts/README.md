# drafts/ ― 記事の「オリジナル（論文版）」置き場（サイト非配信）

考察・記事は **2層**でつくる。ここは①。

| 層 | 中身 | 場所 |
|---|---|---|
| **① オリジナル（論文版）** | 厳密・**出典完備**・論証・自説の完全版（source-of-truth） | **このフォルダ** `drafts/<id>.md` |
| **② Web記事（噛み砕き版）** | 一般向けに要約・翻案した公開版 | `data/articles/<id>.md` |

## ルール

- **サイトには配信しない**（`reference/` と同じ編集者用資料。ページから読み込まれない）。
- **1記事1ファイル**：`drafts/<article-id>.md`。② `data/articles/<article-id>.md` と**同じ id で対応づける**（メタに紐付けフィールドは持たせない＝id一致で足りる）。
- **①が②の裏付け**：①で出典を固め自説を論証 → ②で噛み砕いて公開する。①→②の順は「原典で裏取り→噛み砕く」と同じ構造で、これがサイトの差別化（質）を担保する。
- ひな形は [`_template.md`](_template.md)。

## 公開範囲の注意

このリポジトリが **public** の場合、`drafts/` も GitHub 上では見える（サイトには出ないが、コードとしては閲覧可能）。
①（論文版）を**完全に非公開**にしたいなら、次のどちらか：
- リポジトリを **private** にする、または
- `.gitignore` に `drafts/*.md`（`!drafts/README.md`・`!drafts/_template.md` は残す）を足し、**中身はローカルのみ**にする。
